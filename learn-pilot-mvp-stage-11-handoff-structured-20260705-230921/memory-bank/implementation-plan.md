# 基础产品 MVP 实施计划

## 0. 范围声明

根据 `AI学习计划App_PRD.md` 和 `tech-stack.md`，当前产品是一款移动端优先的 AI 学习计划 App。这里的“基础产品 MVP”聚焦最小可用学习闭环：

```text
创建学习目标 -> 基础测评 -> AI 生成学习路线 -> 查看今日任务 -> 完成任务 -> 查看进度 -> 记录笔记 -> 基础复盘
```

MVP 阶段不实现：

- 3D / 游戏引擎
- WebSocket 实时通信
- 多人协作
- 社交系统
- 排行榜
- 成就系统
- 付费订阅
- 商业化运营y
- 复杂后台管理
- 全网资源爬取
- 原生 App 上架

必须遵循技术栈：

```text
Next.js + React + TypeScript
Tailwind CSS + shadcn/ui
PostgreSQL + Prisma
Zod
OpenAI API through server-side AI service
Auth.js or Supabase Auth
Vitest + React Testing Library + Playwright
Vercel + Supabase/Neon + Sentry
```

## 1. MVP 验收目标

基础 MVP 完成后，用户应能完成以下闭环：

1. 打开移动端 Web App。
2. 创建一个学习目标。
3. 完成或跳过基础测评。
4. 生成一份结构化学习路线。
5. 查看今日任务。
6. 完成任务并看到进度变化。
7. 查看资源建议。
8. 记录一条简单笔记。
9. 进入基础复盘页，看到复盘记录和待确认的 AI 调整建议入口。

## 2. AI 开发执行规则

AI 开发者在每个开发步骤开始前，必须先阅读：

- `AI学习计划App_PRD.md`
- `tech-stack.md`
- `memory-bank/@architecture.md`，如果存在
- `memory-bank/@product.md`，如果存在
- `memory-bank/@ai-rules.md`，如果存在
- 当前要修改模块附近的 schema、types、tests

必须遵守：

- 页面文件只负责路由、布局和组合组件。
- 复杂业务逻辑必须放到 feature、service、repository 或 AI task。
- 前端组件不得直接访问数据库、AI SDK、密钥或服务端环境变量。
- 所有数据库访问必须经过 repository 或 service。
- 所有 AI 调用必须经过 `src/server/ai/`。
- 所有 AI 输出必须使用 Zod schema 校验。
- Prompt 必须独立成文件，不得写在 UI 组件里。
- AI 调整建议默认是“待确认”，用户确认后才可生效。
- 禁止单体巨文件。

## 5. 文件拆分红线


| 对象             | 建议上限 | 超过后处理方式                                 |
| ---------------- | -------- | ---------------------------------------------- |
| 单个 React 组件  | 180 行   | 拆分展示组件、表单组件、列表项组件、状态组件   |
| 单个业务 service | 250 行   | 按 use case 拆分 service 或提取纯函数          |
| 单个工具文件     | 150 行   | 按领域拆分，例如 dates、formatters、validators |
| 单个 prompt 文件 | 200 行   | 拆分系统规则、输出格式、示例                   |
| 单个测试文件     | 250 行   | 按场景拆分                                     |
| 单个 API route   | 80 行    | 把业务逻辑移入 service                         |

禁止模式：

- `components/App.tsx` 包含整个产品。
- `page.tsx` 同时包含表单、API 调用、AI prompt、数据库查询和所有 UI。
- `utils.ts` 存放所有无关函数。
- `ai.ts` 存放所有 prompt 和所有模型调用。
- `types.ts` 存放全项目所有类型。
- `service.ts` 处理所有业务模块。

推荐模式：

- 一个页面组合多个 feature。
- 一个 feature 自己维护组件、schema、types 和 actions。
- 一个 service 只处理一个业务领域。
- 一个 AI task 只处理一种模型能力。
- 一个 prompt 文件只服务一个明确任务。

## 6. MVP 完成定义

基础产品 MVP 可以交付的标准：

- 用户能完成一条完整学习计划闭环。
- AI 调用只发生在服务端。
- AI 输出有 schema 校验。
- AI 调整建议默认待确认。
- 数据能持久化到 PostgreSQL。
- 页面移动端可用。
- 核心路径有 Playwright 测试。
- 业务逻辑被拆分到 feature、service、repository、AI task。
- 没有单体巨文件。
- 没有 3D、WebSocket、排行榜、成就、社交、商业化等非 MVP 内容。
